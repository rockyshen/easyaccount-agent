# 【已废弃】WebSocket 聊天登录鉴权（user-center）

> **本方案已废弃。**  
> 决策变更：不再调用 my-api / my-biz-platform user-center。  
> **请改看**：[本地user表登录与多用户隔离.plan.md](本地user表登录与多用户隔离.plan.md)

以下内容为历史存档，勿再按此实施。完整原文可通过 git 历史查看。

## 原目标（作废）

在 WebSocket 握手阶段通过 HTTP 调用 my-biz-platform Gateway 的 `GET /api/user/current` 校验 JWT。

## 废弃原因

- 改为使用本库 `user` 表本地登录
- 需要真正的多用户账本隔离（`account`/`flow.user_id`）
- 需要首次登录后长期会话免重复输密（本地 `auth_token`）
